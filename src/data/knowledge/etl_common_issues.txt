【ETL 延迟】
现象：结果表未按预期时间更新，报表或监控指标整体滞后。
常见原因：调度失败、资源不足、上游表延迟、任务依赖未完成。
排查步骤：
1. 检查调度任务状态
2. 检查任务日志
3. 检查上游依赖表是否延迟
4. 检查集群资源是否不足
处理建议：优先确认最近一次失败任务、上游依赖和调度状态。

【字段同步失败】
现象：某字段为空、全空或明显缺失，导致下游统计异常。
常见原因：字段映射错误、同步规则缺失、schema 变更、接口返回异常。
排查步骤：
1. 检查字段映射配置
2. 检查同步日志
3. 检查源表字段是否正常
4. 检查表结构或接口返回是否变化
处理建议：优先排查最近上线变更、字段映射和 schema 调整。

【字段缺失率异常】
现象：某字段缺失率突然升高。
常见原因：上游采集异常、埋点缺失、同步失败、字段口径变更。
排查步骤：
1. 检查上游表是否延迟
2. 检查采集链路是否中断
3. 对比历史分布
4. 确认字段口径是否有变更
处理建议：优先检查最近一次上线、埋点改动、ETL 任务状态和同步日志。

【口径切换】
现象：同一指标在不同报表、不同时间点之间出现明显差异。
常见原因：指标定义变更、过滤条件变化、时间窗口变化、统计口径未统一。
排查步骤：
1. 核对指标定义
2. 检查过滤条件
3. 核对时间窗口
4. 对比口径变更记录
处理建议：先统一口径，再比较指标结果。

【源表延迟】
现象：下游结果表整体缺失或更新时间滞后。
常见原因：源表任务失败、调度阻塞、资源不足、上游接口延迟。
排查步骤：
1. 检查源表产出时间
2. 检查上游调度状态
3. 检查是否有依赖阻塞
4. 检查最近异常日志
处理建议：优先确认源表任务是否成功，再看下游链路。

【数据链路中断】
现象：数据在某个中间层消失，导致最终表为空或部分字段缺失。
常见原因：同步任务失败、消息堆积、接口中断、任务异常退出。
排查步骤：
1. 检查链路每一层的输入输出行数
2. 检查同步任务状态
3. 检查消息积压或接口异常
4. 确认链路中最近一次成功时间
处理建议：先定位断点在哪一层，再回看该层日志。

【报表未更新】
现象：BI 看板或报表数据未刷新，指标停留在旧日期。
常见原因：报表缓存未刷新、下游取数失败、结果表未更新、调度延迟。
排查步骤：
1. 检查结果表是否更新
2. 检查报表任务是否成功
3. 检查缓存刷新逻辑
4. 检查取数 SQL 或接口是否报错
处理建议：先判断是结果表问题还是展示层问题。

【一致性问题】
现象：同一业务指标在不同表、不同报表或不同看板之间不一致。
常见原因：指标口径不同、过滤条件不同、ETL 链路异常、时间窗口不一致。
排查步骤：
1. 核对指标口径定义
2. 检查过滤条件
3. 核对时间窗口
4. 检查源表到结果表的数据链路
处理建议：先统一口径，再逐层核对链路。

【分布漂移】
现象：字段分布与历史相比出现明显变化，可能影响模型稳定性。
常见原因：渠道变化、人群结构变化、特征加工逻辑改动、时间窗口变化。
排查步骤：
1. 对比近期与历史分位数
2. 检查渠道和人群结构
3. 核对特征加工逻辑
4. 评估对模型效果的影响
处理建议：先定位分布变化来源，再判断是否影响模型表现。

【最近上线后指标异常】
现象：系统上线后多个字段或指标同时波动。
常见原因：埋点变更、字段映射变化、ETL 配置改动、统计口径调整。
排查步骤：
1. 检查最近上线记录
2. 核对埋点与字段映射
3. 检查 ETL 配置变更
4. 对比上线前后样本分布和结果差异
处理建议：优先检查最近版本改动与变更记录。
